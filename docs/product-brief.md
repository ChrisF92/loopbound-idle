# Loopbound Idle: Kingdom - Product Brief

## High concept

Loopbound Idle: Kingdom is an idle kingdom builder where every reign ends in the
same looping catastrophe. The player repeatedly grows a kingdom, learns from the
collapse, and starts the next age with permanent legacy advantages.

The game should feel like a compact strategy idle: mostly menus, numbers,
milestones, challenges, logs, and build presets. It should avoid art-heavy
systems that would slow down a first Android release.

## Player fantasy

The player is not a single hero. They are the memory of the kingdom itself:
laws, monuments, archives, rituals, and institutions survive each loop even when
the age resets.

## Core loop

1. Gather basic resources.
2. Build production structures.
3. Research upgrades and kingdom laws.
4. Push toward collapse goals.
5. Collapse the age to gain Legacy.
6. Spend Legacy on permanent improvements.
7. Attempt challenge reigns with special restrictions.

## Launch positioning

The game should be marketed as:

> A strategic idle kingdom builder built around time loops, prestige, and
> challenge runs.

## Target platform

Primary target: Android / Google Play.

Secondary target: desktop builds for testing and balancing.

## Design constraints

- UI-first, portrait-friendly.
- Offline progress is important.
- No multiplayer dependency.
- No server dependency for MVP.
- All paid features must be convenience, cosmetic, or optional-ad based.
- Any progression-affecting power must be earnable through normal play.

## Initial content targets

For a first public release, aim for:

- 6-8 resources.
- 8-12 buildings.
- 30-50 upgrades.
- 10-15 challenge reigns.
- 1 main prestige layer.
- 1 deeper meta layer only if the first layer is stable and fun.
- Export/import saves before cloud sync.

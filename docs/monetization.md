# Monetization Guardrails

Loopbound Idle: Kingdom should use a freemium model without selling exclusive
progression power.

## Core rule

Anything that affects permanent player power must be earnable through normal
play. Purchases can reduce friction, improve UX, unlock cosmetics, or provide
optional ad-supported convenience.

## Acceptable paid features

- Remove ads.
- Extra build preset slots.
- Extra automation profile slots.
- Additional save slots.
- Cosmetic UI themes.
- Custom kingdom banners or crest cosmetics.
- Advanced statistics dashboards.
- Run history filters.
- Offline report detail views.
- Optional advisor/hint tools.

## Acceptable rewarded ad uses

Rewarded ads should be opt-in and capped.

Possible rewards:

- Claim a temporary offline-report bonus with a daily cap.
- Trial an extra preset slot for a limited period.
- Receive a non-exclusive convenience consumable.
- Speed up a tutorial-era wait that is not part of competitive balance.

## Avoid

- Selling permanent production multipliers.
- Selling exclusive buildings, challenges, laws, or resources.
- Selling uncapped offline progress.
- Selling premium-only automation that gives stronger progression.
- Selling a premium currency that can buy raw power.
- Creating intentionally painful UX solely to sell relief.

## Recommended first product

Start with one purchase:

**Royal Ledger Pack**

- Removes forced/interstitial ads if any are added later.
- Adds extra build preset slots.
- Adds extra automation profile slots.
- Unlocks additional UI themes.
- Adds advanced statistics panels.

This creates a clear value proposition for invested players while preserving a
fair progression model.
